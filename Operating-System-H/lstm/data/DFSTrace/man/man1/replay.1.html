NAME

replay -- trace prettyprinter

SYNOPSIS

replay [ -v ] [ -d ] [ -l ] [ -f filterfile ] file

DESCRIPTION

  replay reads and pretty-prints the trace in file. The default is an abbreviated format of one record per line. Using the -l switch an exhaustive printout of every field in every record can be produced.

OPTIONS

-d  Prints library debugging information.  Lots of it.

-f filterfile  Apply the filter in filterfile to the trace.

-l  Long-form output of each record.

-v  Verbose mode.  Prints information on what the library is doing.

DIAGNOSTICS

The following are frequently encountered errors:

can't process file file -- the file wasn't there or if a filter file was specified, it couldn't be applied.
